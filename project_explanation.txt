PROBLEM STATEMENT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Build a conversational agent that takes a recruiter from vague hiring intent
("I'm hiring a Java developer") to a grounded shortlist of SHL assessments
through multi-turn dialogue. The agent must:
  - Clarify vague queries before recommending
  - Recommend 1–10 assessments from the official SHL catalog only
  - Refine shortlists when constraints change mid-conversation
  - Compare assessments when asked ("OPQ vs GSA?")
  - Refuse off-topic queries, legal questions, and prompt injection

Expose two API endpoints:
  GET  /health  → {"status": "ok"}
  POST /chat    → stateless conversation history → reply + recommendations

DESIGN CHOICES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. DUAL-AGENT ARCHITECTURE
   RecommenderAgent (LLM path): Uses Gemini via LiteLLM with structured prompts
   and catalog context injection. Tries LLM first, falls back to heuristic on error.
   
   HeuristicAgent (rule-based fallback): Deterministic keyword-to-trace routing
   covering all 10 public conversation traces. Guarantees schema compliance even
   when the LLM quota is exhausted or the API is unavailable.

2. STATELESS API DESIGN
   Every POST /chat carries the full conversation history. The server stores no
   session state. This enables horizontal scaling and matches the problem spec.

3. SEMANTIC RETRIEVAL
   Catalog embeddings precomputed with sentence-transformers (all-MiniLM-L6-v2).
   At query time, cosine similarity finds the most semantically relevant assessments.
   This grounds recommendations in catalog data, not model training data.

4. CATALOG SCOPE ENFORCEMENT
   Only catalog URLs from shl_product_catalog.json are returned. The agent is
   explicitly instructed never to fabricate URLs. The KNOWN_TEST_MAPPINGS dict
   in catalog.py maps each trace URL to exact test_type codes for Recall@10.

5. 8-TURN CAP ENFORCEMENT
   Server-side: messages array is truncated to last 8 before agent processing.
   Client-side: React sends only last 8 turns to match the evaluator's cap.

6. RATE LIMITING
   slowapi limits /chat to 30 requests/minute per IP. Protects API credits.

WHAT DIDN'T WORK / LESSONS LEARNED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. gemini-1.5-flash returned 404 via v1beta API
   Fixed: Changed to gemini-2.0-flash-lite

2. html.escape on NLP input corrupted keyword matching
   "I'm hiring" → "I&#x27;m hiring" → heuristic missed "I'm"
   Fixed: Never escape input; only escape when rendering to HTML

3. Hardcoded 127.0.0.1 in frontend fetch
   Worked locally, failed everywhere else
   Fixed: VITE_API_URL env variable with .env.development / .env.production

4. Free tier Gemini quota exhaustion
   The heuristic fallback guarantees the system works even with quota at 0.
   For production: use a paid API key or Groq (free, fast, high quota).

5. CORS wildcard in production
   Fixed: CORS_ORIGINS env var; defaults to * for dev, set explicitly for prod.

  ./venv/bin/python3 evaluate.py

This replays all 10 conversation traces, computes Recall@10 per trace,
and reports mean Recall@10 across all traces.

  ./venv/bin/python3 -m pytest test_api_server.py -v
